# G.TYP.BOL.03 && 和 || 操作符的右侧操作中不应包含副作用

## 级别
建议

## 规范描述
逻辑与（&&）、逻辑或（||）表达式中的右侧操作是否被执行，取决于左操作的求值结果，当左操作的求值结果可以得出整个逻辑表达式的结果时，不会再计算右操作的结果。

如果右操作包含副作用，则不能确定是否确实发生了副作用。

## 说明
副作用是指对执行状态产生影响，包括：修改对象、修改文件等。在某些情况下副作用会给程序带来不必要的麻烦，降低程序的可读性，其产生的错误也难以查找。

## 检查要点
- `&&` 和 `||` 右侧是否有函数调用（可能有副作用），如文件操作、网络请求、数据库操作等
- `&&` 和 `||` 右侧是否有赋值操作（`=`、`+=`、`-=` 等）
- `&&` 和 `||` 右侧是否有 IO 操作（如 `println!`、`write!`、`fs::` 系列函数等）
- `&&` 和 `||` 右侧是否有修改可变引用或可变变量的操作
- `&&` 和 `||` 右侧是否有 `Cell`/`RefCell` 等内部可变性类型的修改操作

## 反例

```Rust
// 不符合
if flag > 0 || fs::remove_file("example.txt").is_ok() {
  // ...
}
```

## 正例

```Rust
// 符合
if flag > 0 {
  // ...
} else {
    let removed = fs::remove_file("example.txt").is_ok(); 
    if removed {
    // ...
  } 
}
```

## 检查指令
扫描项目中所有 `.rs` 文件，执行以下检查：

1. 定位所有包含 `&&` 和 `||` 操作符的表达式。
2. 分析操作符右侧的表达式内容：
   - 检查是否包含函数调用（特别是已知有副作用的函数，如 `fs::remove_file`、`fs::write`、`println!`、`eprintln!`、`write!` 等）。
   - 检查是否包含赋值操作（`=`、`+=`、`-=`、`*=`、`/=` 等）。
   - 检查是否包含对 `Cell::set`、`RefCell::borrow_mut`、`Mutex::lock` 等内部可变性操作。
   - 检查是否包含 `push`、`pop`、`insert`、`remove` 等修改集合的方法调用。
3. 对于纯函数调用（如 `.is_empty()`、`.len()`、比较操作等），不标记为问题。
4. 对于可能有副作用的操作，标记为"&& / || 右侧表达式可能包含副作用"，并说明具体的副作用类型。
5. 汇总所有不符合项，按文件路径和行号列出问题。
